A **topic** in Novu is a named group of subscribers. Instead of looping through individual user IDs and triggering a workflow for each one, you add subscribers to a topic and trigger the workflow once — Novu fans out a separate notification event to every subscriber in the group. This makes topics the right tool for any scenario where the same message needs to reach multiple people at once.

## Topic identifiers

Each topic is uniquely identified by a **`topicKey`** — a permanent, internal reference you define. You can also assign a human-readable **name** for display in the dashboard (e.g., `"Task Updates"` or `"Comment Thread Subscribers"`).

The `topicKey` must be unique within an environment and cannot be changed after creation. Common patterns:

- `task:taskId` — notifications for a specific task
- `post:postId` — notifications for a specific post
- `team:teamId` — notifications for all members of a team
- `feature-release` — broadcast for a product announcement

<Note>
  Topic keys must be unique per environment. You can reuse the same key (e.g., `feature-release`) across different environments like development and production.
</Note>

## How topics fit into Novu's model

Novu's delivery model is built around workflows triggered for one or more recipients. Topics act as a special recipient type representing a group rather than an individual.

When you trigger a workflow to a topic:

1. Novu looks up all subscribers assigned to that topic.
2. A **separate workflow event** is created for each subscriber.
3. Each subscriber's preferences, channel credentials, and personalization data are applied independently.

This means every subscriber in a topic still gets a personalized, preference-respecting notification — topics just remove the overhead of managing individual trigger calls.

## Scalability

Topics are designed for high-volume use:

- Each topic supports up to **100,000 subscribers**.
- Triggering a workflow to a topic creates one event per subscriber.
- Triggering a workflow to an empty topic is a no-op and does not count toward billing.

## Autogenerated topics

If you call the "add subscribers" API with a `topicKey` that doesn't exist yet, Novu automatically creates a new topic named `Autogenerated-<TOPIC_KEY>` and assigns the subscribers to it. You can rename this topic from the dashboard afterward.

## Triggering workflows to topics

### Trigger to a single topic

Use the `to` field with `type: "Topic"` and your `topicKey`:

```typescript
import { Novu } from "@novu/api";

const novu = new Novu({ secretKey: "<YOUR_SECRET_KEY_HERE>" });

await novu.trigger({
  workflowId: "workflow_identifier",
  to: { type: "Topic", topicKey: "topic_key" },
});
```

### Trigger to multiple topics

Pass an array of topic objects to notify multiple groups in one call:

```typescript
import { Novu } from "@novu/api";

const novu = new Novu({ secretKey: "<YOUR_SECRET_KEY_HERE>" });

await novu.trigger({
  workflowId: "workflow_identifier",
  to: [
    { type: "Topic", topicKey: "topic_key_1" },
    { type: "Topic", topicKey: "topic_key_2" },
  ],
});
```

### Exclude a subscriber (actor)

By default, every subscriber in the topic receives the notification. Use the `actor` field to exclude the subscriber who performed the action — for example, to avoid notifying a user about their own comment:

```typescript
import { Novu } from "@novu/api";

const novu = new Novu({ secretKey: "<YOUR_SECRET_KEY_HERE>" });

await novu.trigger({
  workflowId: "workflow_identifier",
  to: [{ type: "Topic", topicKey: "topic_key_1" }],
  actor: "actor_subscriber_Id",
});
```

## Dynamic group membership

You can add or remove subscribers from a topic at any time. The topic always reflects the current state of your application's relationships. Examples:

- Add a user to `task:123` when they're assigned to a task; remove them when unassigned.
- Add a user to `team:456` when they join the team; remove them when they leave.

Subscribers don't need to know they belong to a topic, and membership never affects their personal notification preferences.

## Frequently asked questions

<AccordionGroup>
  <Accordion title="Do topics override subscriber preferences?">
    No. Topics only define delivery groups. Each subscriber's individual notification preferences remain fully intact.
  </Accordion>
  <Accordion title="Can a subscriber belong to multiple topics?">
    Yes. A subscriber can be a member of any number of topics simultaneously, enabling overlapping targeting strategies.
  </Accordion>
  <Accordion title="Is a topic like a mailing list?">
    Conceptually yes — topics group users to receive shared messages. The key difference is that topics are tightly integrated with your application logic and are much more dynamic than a traditional mailing list.
  </Accordion>
  <Accordion title="What happens if I trigger a workflow to an empty topic?">
    The workflow is processed, but no notifications are delivered. Triggers to empty topics do not count toward billing.
  </Accordion>
</AccordionGroup>
