Syncing registers your code-defined workflows with the Novu Cloud dashboard, making them available for triggering via the Novu API and visible to non-technical teammates who manage controls. Novu Framework operates on a **GitOps model** — your codebase is the source of truth, and syncing pushes a snapshot of your workflow definitions to the environment of your choice.

## Novu Environments

Novu gives you three environments to work with:

| Environment | Purpose |
|---|---|
| **Local Studio** | Development only. Runs against your local machine for live workflow previewing and editing. |
| **Development** | Staging environment. Sync here to let teammates review and test changes before production. |
| **Production** | Live environment. Sync here to enable triggering workflows for real subscribers. |

<Note>
  Each environment has its own set of API keys and workflow state. Always sync to the correct environment to avoid accidentally publishing draft changes to production.
</Note>

## Syncing with the CLI

Use the `npx novu@latest sync` command to sync your workflows from the command line. Provide the URL of your deployed [Bridge endpoint](/framework/endpoint) and your Novu secret key:

<CodeGroup>
  ```bash US Region
  npx novu@latest sync \
    --bridge-url https://your-app.com/api/novu \
    --secret-key <NOVU_SECRET_KEY> \
    --api-url https://api.novu.co
  ```

  ```bash EU Region
  npx novu@latest sync \
    --bridge-url https://your-app.com/api/novu \
    --secret-key <NOVU_SECRET_KEY> \
    --api-url https://eu.api.novu.co
  ```
</CodeGroup>

Replace `https://your-app.com/api/novu` with the actual public URL of your Bridge endpoint. For example:

- **Deployed app** — `https://api.your-domain.com/api/novu`
- **Local tunnel** — `https://<UUID>.novu.sh/api/novu` (generated by `npx novu@latest dev`)

<Tip>
  Store `NOVU_SECRET_KEY` as a CI/CD environment secret rather than passing it inline. Reference it as `$NOVU_SECRET_KEY` in your pipeline scripts.
</Tip>

## What Syncing Does

When you run `sync`, Novu:

1. Calls your Bridge endpoint to discover all registered workflows
2. Validates the workflow definitions, schemas, and step configurations
3. Registers (or updates) the workflows in the target environment
4. Makes the workflows available for triggering via the Novu API and dashboard

After a successful sync, your workflows appear in the Novu dashboard under the **Workflows** tab. Non-technical teammates can then open any step and edit its controls.

## GitOps Workflow

The recommended flow for shipping workflow changes follows standard Git branching:

<Steps>
  <Step title="Create a feature branch">
    Branch off from your main development branch and build your workflow changes locally, testing with the Novu Local Studio.
  </Step>
  <Step title="Sync to Development">
    Once ready, deploy your branch to your staging environment and sync:

    ```bash
    npx novu@latest sync \
      --bridge-url https://staging.your-app.com/api/novu \
      --secret-key $NOVU_DEV_SECRET_KEY
    ```
  </Step>
  <Step title="Test end-to-end">
    Verify the workflows behave correctly in the Development environment. Have teammates review and adjust controls if needed.
  </Step>
  <Step title="Merge to main and sync to Production">
    After approval, merge to your main branch. Your CI/CD pipeline syncs to the Production environment automatically (see below).
  </Step>
</Steps>

## CI/CD Integration

Automate syncing as part of your deployment pipeline so workflows are always in sync with your deployed code.

### GitHub Actions

Use Novu's [direct GitHub Actions integration](https://docs.novu.co/framework/deployment/actions) for the simplest setup.

### Other CI/CD Providers

For any other CI provider, add a step that runs the CLI sync command after your application deploys. Here is a generic example:

```yaml
# Example: run after deployment step
- name: Sync Novu workflows
  run: |
    npx novu@latest sync \
      --bridge-url $BRIDGE_URL \
      --secret-key $NOVU_SECRET_KEY
  env:
    BRIDGE_URL: https://your-app.com/api/novu
    NOVU_SECRET_KEY: ${{ secrets.NOVU_SECRET_KEY }}
```

## Using Vercel Preview URLs

On Vercel's free tier, preview deployment URLs are not publicly accessible by default. To allow Novu to reach your preview endpoint, enable [Protection Bypass for Automation](https://vercel.com/docs/security/deployment-protection/methods-to-bypass-deployment-protection/protection-bypass-automation) in your Vercel project settings and append the bypass secret as a query parameter:

```bash
npx novu@latest sync \
  --bridge-url "https://my-app-preview.vercel.app/api/novu?x-vercel-protection-bypass=BYPASS_SECRET" \
  --secret-key $NOVU_SECRET_KEY
```
