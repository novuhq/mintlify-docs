Novu's `@novu/nextjs` package gives you a ready-made `<Inbox />` component that renders a real-time notification bell directly in your Next.js layout. Follow this guide to go from zero to a working notification inbox in under five minutes.

## Prerequisites

- A [Novu account](https://dashboard.novu.co/auth/sign-up) — create one for free or [sign in](https://dashboard.novu.co/auth/sign-in) if you already have one.
- Node.js 18+ and a Next.js project (App Router recommended).

## Setup

<Steps>

### Create a Next.js application

If you don't have an existing project, scaffold one now:

<CodeGroup>
```bash npm
npm create next-app@latest
```
```bash pnpm
pnpm create next-app@latest
```
```bash yarn
yarn create next-app
```
```bash bun
bun create next-app
```
</CodeGroup>

### Install `@novu/nextjs`

Add the Novu Next.js SDK to your project:

<CodeGroup>
```bash npm
npm install @novu/nextjs
```
```bash pnpm
pnpm add @novu/nextjs
```
```bash yarn
yarn add @novu/nextjs
```
```bash bun
bun add @novu/nextjs
```
</CodeGroup>

### Add the Inbox component to your layout

Open (or create) `app/layout.tsx` and import the `<Inbox />` component from `@novu/nextjs`. Place it inside your navbar so it's available on every page:

```tsx app/layout.tsx
import { Inbox } from '@novu/nextjs';
import './globals.css';

export const metadata = {
  title: 'Novu Next.js Quickstart',
  description: 'Generated by create next app',
};

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        <nav className="flex justify-end items-center p-4 gap-4 h-16">
          <Inbox
            applicationIdentifier="YOUR_APPLICATION_IDENTIFIER"
            subscriber="YOUR_SUBSCRIBER_ID"
          />
        </nav>
        {children}
      </body>
    </html>
  );
}
```

<Note>
  Replace `YOUR_APPLICATION_IDENTIFIER` with the value from **API Keys** in your Novu dashboard, and `YOUR_SUBSCRIBER_ID` with your user's unique ID. If you pass a `subscriberId` that doesn't exist yet, Novu creates that subscriber automatically.
</Note>

**Finding your credentials:**

- **`applicationIdentifier`** — In the [Novu dashboard](https://dashboard.novu.co), click **API Keys** and copy your Application Identifier.
- **`subscriberId`** — This represents a user in your system, typically their database user ID. For testing, use the auto-generated ID shown under the **Subscribers** tab in the dashboard.

### Run your application

Start your development server:

<CodeGroup>
```bash npm
npm run dev
```
```bash pnpm
pnpm dev
```
```bash yarn
yarn dev
```
```bash bun
bun dev
```
</CodeGroup>

Once running, you'll see a **bell icon** in the top-right corner of the navbar. Click it to open the notification inbox — it's empty for now, so let's trigger a notification.

### Trigger your first notification

Create a workflow in the Novu dashboard and send a test notification:

1. Go to your [Novu dashboard](https://dashboard.novu.co).
2. Click **Workflows** in the sidebar.
3. Click **Create Workflow** and enter a name (e.g., `Welcome Notification`).
4. Click **Create Workflow** to save it.
5. Click the **Add Step** icon in the workflow editor and select **In-App** as the step type.
6. In the In-App template editor, set:
   - **Subject**: `Welcome to Novu`
   - **Body**: `Hello, world!`
7. Close the editor, then click **Trigger**.
8. Click **Test Workflow**.

### View the notification in your app

Switch back to your Next.js app and click the bell icon.

You should see the notification you just triggered from the dashboard. 🎉

</Steps>
